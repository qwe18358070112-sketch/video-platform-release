## 本轮覆盖的问题类型

这轮冻结版已经覆盖的核心问题包括：

- 全屏固定宫格识别误判
- `F2 / F8 / F9` 热键恢复链路错误
- 连续步进后的恢复判定失误
- 同进程前台辅窗误判导致的运行时守卫重跑
- 全屏 `4/6/9/12` 模式化链路的统一收口

## 后续变更规则

冻结后按下面规则维护：

1. 默认只修新增问题，不再主动扩改已通过链路。
2. 非全屏链路默认不主动调整，除非出现明确回归。
3. 任何后续修改都必须至少覆盖：
   - 对应布局
   - 对应模式
   - 一次完整轮巡
   - `F2 -> F2`
   - `F2 -> F9 x6 -> F2`
   - `F2 -> F8 -> F2`
4. 不接受只靠现场按键恢复的临时补法，必须补回归测试。
