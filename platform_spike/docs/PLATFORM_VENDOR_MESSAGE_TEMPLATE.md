# 发给平台方的确认消息模板

下面这段可以直接发给海康平台方、实施方或项目集成方。

---

我们准备把当前“视频融合赋能平台客户端自动轮询”方案，从桌面客户端自动化迁移成“平台能力驱动的受控预览运行时”。  
为了进入 POC 开发阶段，请帮忙确认以下能力与交付物：

## 1. OpenAPI / 资源查询能力

请确认当前部署环境是否开放以下接口：

1. 组织/区域/监控点/设备/通道查询
2. 监控点与真实设备通道映射
3. 实时预览能力
4. 录像回放能力
5. 平台鉴权接口或 AppKey/AppSecret 申请方式

## 2. Web 插件 / WebControl 能力

请确认当前环境是否支持官方视频 WEB 插件，并提供：

1. 插件版本与安装包
2. 开发文档或 demo
3. 前端接入所需 js 文件与运行依赖
4. 是否支持自建页面承载预览
5. 是否支持 4 / 6 / 9 / 12 宫格切换
6. 是否支持选中某一路并放大/退出放大
7. 是否支持这些动作的事件回调或消息通知

## 3. 鉴权参数

如果走插件路线，请明确是否使用以下参数模型：

- appkey
- appsecret
- 平台 ip
- https 端口
- authUuid

并请提供测试环境所需的：

- 测试平台地址
- 测试账号或鉴权参数申请方式
- 测试组织/监控点/设备资源

## 4. 预览承载路线建议

请明确建议我们走哪条路线：

1. 平台 Web 插件预览
2. 平台 OpenAPI + 前端播放器
3. 设备侧 HCNetSDK 直连
4. 平台中转流地址

## 5. 我们的最小 POC 目标

平台方确认后，我们只做一个最小闭环：

1. 打开 4 宫格
2. 切到 9 宫格
3. 放大其中一路
4. 返回宫格

所以请优先回复和上面 4 个动作直接相关的能力与文档。
