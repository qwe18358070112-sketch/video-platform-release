# 平台方确认清单

这份清单用于向海康平台方或项目实施方确认，当前项目是否可以从“桌面客户端自动化”迁移到“平台能力驱动的受控预览”。

## 已通过公开资料确认的事项

### 1. Web 插件路线是官方支持能力

海康公开文档已经确认：

- 视频 WEB 插件由 `WebControl.exe` 管理
- 可以通过 `JS_CreateWnd` 创建播放窗口
- 支持 `进入全屏 / 退出全屏` 接口与消息
- 支持 `切换布局消息`
- 支持 `播放窗口双击事件消息`
- 支持多平台接入
- 支持通过 `setAuthInfo` 设置 `appkey / secret / ip / port / authUuid`

这说明理论上可以做“自有页面承载 + 受控布局 + 事件回调”的方案。

### 2. SDK 直连路线也是官方支持能力

HCNetSDK 公开结构体和接口说明已经确认：

- `NET_DVR_PREVIEWINFO` 允许把预览绑定到窗口句柄 `hPlayWnd`
- `NET_DVR_RealPlay` 可发起预览
- `NET_DVR_GetRealPlayerIndex` 可拿到底层播放器句柄，用于扩展抓图等能力

这说明理论上可以绕过桌面客户端，自己做受控预览窗格。

## 必须由平台方确认的私有项

下面这些问题不能从公开文档里最终确认，必须由你们平台方、实施方或拥有平台控制台权限的人明确回复。

### A. OpenAPI / 平台接口清单

请确认当前部署的“视频融合赋能平台”是否开放了以下能力：

1. 组织/区域/监控点/设备/通道查询接口
2. 监控点与真实设备通道的映射接口
3. 实时预览相关接口
4. 录像回放相关接口
5. 平台鉴权接口或 AppKey/AppSecret 申请方式
6. 是否支持第三方前端页面嵌入预览能力

### B. Web 插件接入方式

请确认当前环境是否支持官方视频 WEB 插件，并明确：

1. 使用的是哪一代插件
2. 对应的插件安装包和开发包版本
3. 前端接入所需文件清单
4. 是否要求浏览器、WebView2 或特定内核
5. 是否允许在项目自建页面中加载插件
6. 是否支持布局切换、全屏、窗口选中、双击事件回调
7. 是否支持按 `cameraIndexCode` 或等价资源 ID 打开预览

### C. 鉴权方式

请确认前端/插件接入使用哪一种鉴权模型：

1. `appkey + secret + ip + port`
2. 单点登录后的平台 session
3. 反向代理统一鉴权
4. 其他私有鉴权方式

如果是官方文档里的 `setAuthInfo` 路线，请明确：

1. `appkey`
2. `appsecret`
3. 平台网关 `ip`
4. HTTPS 端口 `port`
5. 是否需要平台公钥加密字段
6. `authUuid` 使用规范

### D. 资源查询能力

请确认是否有接口能直接返回下面这些信息：

1. 监控点编号 `cameraIndexCode`
2. 监控点名称
3. 所属组织/区域
4. 设备编号
5. 通道编号
6. 在线状态
7. 码流能力

### E. 预览承载方式

请明确项目应走哪条承载路线：

1. 平台 Web 插件预览
2. 平台 OpenAPI + 前端播放器
3. 设备侧 HCNetSDK 直连
4. 平台中转流地址

并回答：

1. 4 / 6 / 9 / 12 宫格是否可由接口直接控制
2. 是否支持选中某一路并放大
3. 是否支持退出放大恢复宫格
4. 这些动作是否都有事件回调或返回值

## 我们需要的平台方最终交付物

至少需要下面这些内容之一：

### 路线 1：Web 插件

1. 开发文档
2. 插件安装包
3. demo 或样例工程
4. 鉴权参数申请方式
5. 资源查询接口文档
6. 监控点预览调用样例

1. SDK 安装包
2. 登录、资源发现、预览相关文档
3. 设备或平台通道号映射说明
4. 测试账号与测试资源
