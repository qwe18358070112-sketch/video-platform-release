# 标定详细操作指南

这份指南专门解决两个问题：

1. 你自己不熟悉怎么标定。
2. 你希望让 Codex 在本机把标定也代做掉。

---

## 一、先理解“标定”到底在做什么

程序不会自己猜“视频宫格到底在客户端窗口的哪一块区域”。

所以标定的本质是：

- 告诉程序**视频预览总区域的左上角**在哪里
- 再告诉程序**视频预览总区域的右下角**在哪里

程序再根据这块总区域，按 4 / 6 / 9 / 12 宫格去切分每一路的点击区域。

也就是说，你标定的不是“某一个小格子”，而是：

**整个视频宫格大区域**。

---

## 二、什么时候必须重新标定

下面任意一种情况发生，都建议重新标定：

1. 换了一台电脑
2. 分辨率变了
3. 缩放比例变了
4. 客户端版本升级了
5. 非全屏和全屏切换后布局明显不同
6. 发现点击经常点到顶部工具条 / 左侧收藏夹 / 空白区域
7. 改了 `grid.layout_template`

---

## 三、标定前的准备动作

### 1. 先把客户端切到你真正要跑的界面

标定时界面必须已经稳定：

- 非全屏标定：客户端停在**非全屏宫格**
- 全屏标定：客户端停在**全屏宫格**

### 2. 让左侧树和顶部工具条保持你平时真实使用的状态

原因很简单：

- 如果你标定时左侧树展开，运行时也要尽量保持一致
- 如果你标定时顶部工具条在，运行时也尽量保持一致

### 3. 标定前不要先放大某一路

标定必须在**宫格总览态**完成。

---

## 四、非全屏标定：一步一步怎么做

### 命令

```powershell
python app.py --calibrate windowed
```

### 程序会提示你做两次取点

#### 第一次取点：预览总区域左上角

你要把鼠标移动到：

- **整个视频宫格外框**的左上角
- 不是左侧收藏夹
- 不是顶部蓝色标签栏
- 不是单个小格子的内部中心点

然后按一次：

```text
F6
```

#### 第二次取点：预览总区域右下角

你要把鼠标移动到：

- **整个视频宫格外框**的右下角
- 不是右侧空白边缘
- 不是底部状态栏

然后再按一次：

```text
F6
```

程序就会把比例写进 `config.yaml`。

---

## 五、全屏标定：一步一步怎么做

### 命令

```powershell
python app.py --calibrate fullscreen
```

步骤和非全屏完全一样，只是界面换成了全屏宫格。

注意点：

- 全屏下经常更容易把顶部或底部边界估错
- 如果全屏下宫格更贴近屏幕边缘，取点要更精确

---

## 六、标定后必须立刻做的检查

### 检查命令

```powershell
python app.py --inspect-calibration windowed
python app.py --inspect-calibration fullscreen
```

程序会输出两张检查图：

- `tmp/calibration_windowed_inspect.png`
- `tmp/calibration_fullscreen_inspect.png`

### 你检查什么

看蓝框和红点：

1. 蓝框是否只包住视频宫格
2. 左侧收藏夹有没有被包进去
3. 顶部工具条有没有被包进去
4. 底部状态区有没有被包进去
5. 每个格子的中心点是否落在视频内容内部，而不是标题栏和分割线边上

### 判断标准

#### 正确标定
- 蓝框刚好覆盖全部宫格
- 红点大致落在每个小格子的有效视频区域中间偏下位置

#### 错误标定
- 蓝框覆盖了左侧树
- 蓝框覆盖了顶部工具条
- 蓝框只框到了前两排，漏了后两排
- 红点落在顶部标题文字、菜单栏、边框线上

只要发现这些问题，就重新标定。

---

## 七、最容易犯的 8 个标定错误

### 错误 1：把左侧收藏夹也框进去
后果：点击点位整体右偏或左偏，顺序没错但经常点不到视频内容。

### 错误 2：把顶部工具条也框进去
后果：双击可能打到标题栏、按钮区、菜单区。

### 错误 3：只按某一个小格子的边界来框
后果：程序以为整个预览区很小，后面切格全错。

### 错误 4：非全屏和全屏只做了一套
后果：切模式后点位整体错位。

### 错误 5：取点点到了格子内部太靠里
后果：外框被截短，右下角或左上角格子容易点歪。

### 错误 6：标定时界面和运行时界面不一样
后果：运行时点位偏移。

### 错误 7：标定完不检查图
后果：错框带病进入自动点击。

### 错误 8：发现错位后只改顺序，不重做标定
后果：顺序修了，点位仍然错。

## 八、给 Codex 的标定执行规则

如果 Codex 具备本机桌面控制能力，它应该这样做：

1. 先阅读本文件
2. 启动客户端并切到目标宫格页
3. 运行 `python app.py --calibrate windowed`
4. 按提示把鼠标移到预览总区域左上角，按 `F6`
5. 再把鼠标移到预览总区域右下角，按 `F6`
6. 运行 `python app.py --inspect-calibration windowed`
7. 检查检查图是否准确
8. 如果不准确，直接重做，不要带错运行
9. 再重复一遍 fullscreen 标定

Codex 的成功标准不是“命令跑过了”，而是：

- 检查图准确
- 红点落位正确
- 后续动作路径能命中视频格子

---

## 九、标定完成后的最小验收

标定通过后，先不要直接长时间轮询。

先做最小验收：

1. 启动程序
2. 只观察第一路
3. 看是否：
   - 单击选中
   - 双击放大
   - 停留
   - 双击返回
   - 再停留
4. 再看第二路是否点位正确

只要第一路或第二路点位明显不对，就回到标定，不要继续调 F9/F11 逻辑。
