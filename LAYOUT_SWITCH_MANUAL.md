# 宫格切换与自动识别操作手册

这份手册专门回答 4 件事：

1. 程序现在会不会自动识别“全屏 / 非全屏”
2. 程序现在会不会自动识别“当前是 4 / 6 / 9 / 12 宫格”
3. 你手动切宫格时，标准操作应该怎么做
4. 顺序、自定义顺序、收藏夹顺序、人工监管和热键该怎么配合使用

## 1. 自动识别能力现在是什么状态

### 1.1 自动识别全屏 / 非全屏

当前主流程支持自动识别。

判定链路：

- `window_manager.py -> WindowManager.detect_mode()`
- 当 `profiles.active_mode: auto` 或运行时使用 `python app.py --run --mode auto` 时启用

判定原则：

1. 先看是否存在真正的附属渲染面，例如 `VSClient`
2. 再看当前界面还能不能看到非全屏特征控件，例如：
   - `收藏夹`
   - `搜索`
   - `全部收藏`
   - `打开文件夹`
   - `视频监控配置`
3. 最后才用几何覆盖率做兜底

现场含义：

- “窗口最大化”不等于“真正全屏”
- 右上角全屏按钮状态也不能单独当成真实全屏证据
- 只有界面内容结构也符合全屏，程序才应该认定为 fullscreen

### 1.2 自动识别当前宫格数

当前主流程支持自动识别 `4 / 6 / 9 / 12`。

也就是：程序会自动识别当前更像 4 / 6 / 9 / 12 中的哪一种宫格。

判定链路：

- `scheduler.py -> _try_sync_runtime_layout()`
- 当前是 **非全屏** 时，主流程不再打开顶部 `窗口分割` 面板；改成同时参考视觉候选、分隔线结构、宫格几何估计这 3 类信号做闭环判定
- 当前是 **全屏** 时，不再强行打开 `窗口分割`；会直接退回视觉同步或手动锁定，避免把全屏界面误当成非全屏去点
- 如果自动识别不稳定，可以直接用参数或热键把程序锁到当前模式

注意：

- `13` 当前仍属于“可手动切换、可核对”的布局
- 自动轮询主流程仍按 `4 / 6 / 9 / 12` 正式支持，不把 `13` 当正式轮询布局

### 1.3 自动 / 手动锁定是什么关系

- `--mode auto` / `--layout` 不传：程序自动识别
- `--mode windowed|fullscreen`：手动锁定当前运行模式
- `--layout 4|6|9|12`：手动锁定当前宫格模式
- `python app.py --inspect-runtime ...`：只读查看当前模式 / 当前宫格 / 运行档位，不切宫格
- `F1`：切换 `自动识别 / 手动锁定`
- `F7`：切换“程序目标模式” `非全屏 / 全屏`；当前若还是自动，会先切到手动锁定
- `F8`：切换“程序目标宫格” `12 / 9 / 6 / 4`；当前若还是自动，会先切到手动锁定
- 状态浮窗会同步显示：实际模式/宫格、目标模式/宫格、是否已闭环匹配、当前顺序

现在的手动锁定不再是“盲信目标值”：

- `F1` 切到手动时，程序会先连续重识别当前现场，再把观察值锁成手动目标
- 暂停后如果你手工改了全屏/窗口或宫格，恢复前程序会再次核对现场是否与目标一致
- 默认情况下 `F7/F8` 只改目标，不直接切客户端
- 如果配置 `controls.runtime_hotkeys_drive_client_ui: true`，暂停态下的 `F7/F8` 会直接调用切换能力把客户端切到目标状态

## 2. 手动切宫格的标准方法

### 2.1 最推荐的方法：直接用项目命令

不要再从左侧收藏夹树去点 `4个画面 / 6个画面 / 9个画面`。

标准命令：

```powershell
python app.py --switch-layout 4
python app.py --switch-layout 6
python app.py --switch-layout 9
python app.py --switch-layout 12
python app.py --switch-layout 13
```

当前映射固定为：

- `4 -> 平均 / 4`
- `6 -> 水平 / 6`
- `9 -> 平均 / 9`
- `12 -> 其他 / 12`
- `13 -> 其他 / 13`

执行后必须人工看一眼中央宫格，确认真的切换成功，不能只看命令有没有返回 JSON 成功。

### 2.2 你必须点击的真实入口

如果你一定要手工点客户端界面，真实入口是顶部工具栏里的 `窗口分割`。

具体入口是：

1. 顶部工具栏
2. 点击 `窗口分割`
3. 在弹出的分组面板里选择目标布局

不要把左侧收藏夹树当成布局切换入口。

### 2.3 非全屏场景的建议操作顺序

1. 先让客户端停在正常非全屏宫格页面
2. 确认左侧收藏夹、顶部工具栏等界面正常可见
3. 执行：

```powershell
python app.py --switch-layout 12
```

或改成 `9 / 6 / 4`

4. 人工确认中央宫格确实已经切换
5. 再启动自动轮询：

```powershell
python app.py --run --mode auto
```

### 2.4 全屏场景的建议操作顺序

1. 先确认客户端已经处于真正全屏，而不是只是窗口最大化
2. 执行：

```powershell
python app.py --switch-layout 12
```

或改成 `9 / 6 / 4`

3. 人工确认：
   - 中央宫格确实变了
   - 左侧收藏夹树不应继续占据主预览区
4. 再执行：

```powershell
python app.py --run --mode auto
```

如果当前“全屏按钮看起来是勾选的，但界面又像非全屏”，先人工回到非全屏，再重新进入真正全屏后继续。

## 3. 自动轮询前，布局与顺序要怎么配

### 3.1 从左到右、从上到下

```yaml
grid:
  order: row_major
```

### 3.2 从上到下、从左到右

```yaml
grid:
  order: column_major
```

### 3.3 自定义顺序

```yaml
grid:
  order: custom
  custom_sequence: [0, 3, 6, 9, 1, 4, 7, 10, 2, 5, 8, 11]
```

### 3.4 按收藏夹名称顺序

先读取当前可见收藏夹名称：

```powershell
python app.py --dump-favorites
```

再把物理窗格和名称对应好：

```yaml
grid:
  order: favorites_name
  cell_labels:
    0: "大厅柱子口"
    1: "法雨寺停车场"
    2: "千步沙入口"
    3: "码头入口"
```

程序会把左侧收藏夹当前可见顺序映射回物理窗格顺序。

## 4. 人工监管与热键怎么配合

运行过程中保留人工监管。

热键：

- `F1`：切换自动识别 / 手动锁定
- `F7`：切换运行模式目标（非全屏 / 全屏）
- `F8`：切换宫格模式目标（12 / 9 / 6 / 4）
- `F2`：开始 / 暂停 / 继续
- `F9`：暂停时预选下一路，并自动选中下一路，但程序保持暂停
- `F10`：安全停止并退出
- `F11`：紧急恢复到宫格，并重走当前未完成动作路径

当前动作路径合同：

`单击选中 -> 双击放大 -> 停留 N 秒 -> 双击返回 -> 停留 N 秒 -> 下一路`

## 5. 自动检测 / 自动恢复 / 跳过黑屏

当前项目里，自动化不是“只会点鼠标”，还带这些守卫能力：

- 自动识别全屏 / 非全屏
- 自动识别当前是否仍在正确宫格界面
- 自动识别当前更像 4 / 6 / 9 / 12 中的哪一种布局
- 黑屏 / 预览失败时按策略处理
- 前台窗口漂移时自动回焦
- 同进程弹窗时自动尝试关闭
- 界面不对时自动恢复到宫格
- 连续失败达到阈值后自动暂停，避免继续乱点

## 6. 你最常用的现场操作范式

### 范式 A：我只想切宫格，然后自动跑

```powershell
python app.py --switch-layout 6
python app.py --run --mode auto
```

### 范式 B：我想先暂停监管，再人工一步步切下一路

```text
F2 暂停 -> F9 -> F9 -> F9 -> F2 继续
```

### 范式 C：我怀疑当前已经乱了，要强制回宫格

```text
按 F11
```

语义不是“跳过当前路”，而是“回宫格并重走当前未完成路径”。

## 7. 现场排障清单

### 7.1 命令返回成功，但中央宫格没变

先不要继续跑自动轮询。

排查顺序：

1. 当前前台是不是目标客户端
2. 当前是不是被别的弹窗挡住
3. 当前是否真的打开了顶部 `窗口分割` 面板
4. 再次执行 `python app.py --switch-layout ...`
5. 仍不行就留日志和截图

### 7.2 看起来已经全屏，但程序像按非全屏在点

先核对：

- 左侧收藏夹是否仍可见
- 顶部工具栏是否仍占主要预览区域
- 是否只是窗口最大化而不是真全屏

### 7.3 轮询顺序不对

先检查 `config.yaml`：

- `grid.order`
- `grid.custom_sequence`
- `grid.cell_labels`
- `profiles.active_mode`

## 8. 最后的结论

当前正确用法就是两句话：

1. 切宫格优先用 `python app.py --switch-layout ...`，不要走左侧树
2. 自动运行优先用 `python app.py --run --mode auto`，让程序自己识别全屏 / 非全屏和当前宫格
